# Finance Skill

## 描述

`finance` 技能支持根据用户提问，动态生成 SQL 查询语句并最终执行该查询。技能包含以下核心模块：

### 意图分析模块

- 解析用户问题的核心意图，判断是否与数据查询相关。
- 提取关键字（如“成本分摊”、“预算分析”等）、映射表和匹配适配。

### 表结构查询模块

- 动态查询数据库表结构和示例数据。
- 验证字段名称和数据类型，确保生成的 SQL 语句字段正确。

### SQL 构造模块

- 动态组装 SQL 查询模板。
- 根据意图分析的结果，填充函数参数 (如表名、字段名、查询条件)。

### 执行模块

- 将生成的 SQL 查询语句提交至数据库。
- 返回查询结果用于进一步分析和展示。

### 结果分析模块

- 对查询结果进行分析，生成用户可读的结论。
- 包括数据汇总、趋势分析和对比分析。

## 执行工具约束

- 必须先执行 SQL 查询并拿到结果，再给出最终回答；禁止在未查询数据库时直接回答业务结论。
- 优先使用代理内置的 `sql_query` 工具执行 SQL（基于 `pyodbc`）。
- 不要依赖 `sqlcmd` 命令行工具；在 Windows 环境中可能未安装，容易导致 `"sqlcmd 不是内部或外部命令"` 错误。
- 如需调试 SQL，可先生成 SQL 文本，再调用 `sql_query` 执行并查看返回的 JSON 结果。
- 禁止通过 `write_file` 生成临时 Python 脚本再间接执行 SQL；必须在当前流程内直接完成查询与回答。
- 禁止仅返回“生成的 SQL 语句”作为最终答案；最终答案必须包含已执行查询得到的结果摘要。

### 导入与路径约束（防止日志中的模块错误）

- 禁止使用 `from allocation_utils import ...` 这类依赖当前工作目录的导入写法。
- 如需在 Python 中导入 finance 脚本，必须使用完整路径：
  - `from skills.finance.scripts.allocation_utils import generate_alloc_sql`
  - `from skills.finance.scripts.sql_query import run_sql_query`
- 若导入失败，不得绕路写临时脚本；应改为修正导入路径后在同一流程重试。

## 数据库信息

请通过 `.env` 配置数据库连接，不在技能文档中写死连接明文：

- `DB_NAME`: 数据库名称
- `DB_SERVER`: 数据库地址/主机
- `DB_PORT`: 端口号（默认 `1433`）
- `DB_USER`: 用户名
- `DB_PASSWORD`: 密码
- `DB_DRIVER`: 驱动（默认 `SQL Server`）

说明：`sql_query` 工具仅从环境变量读取连接配置；未配置时应返回配置缺失错误并停止回答业务结论。

## 工作流逻辑

### 需要执行的脚本

1. **查询表结构脚本**:
   - 脚本路径: `scripts/query_table_structure.py`
   - 功能: 根据表名查询表结构（包括列名、数据类型）以及示例数据。
   - 示例调用:
     ```python
     from skills.finance.scripts.query_table_structure import get_table_structure
     structure, sample_data = get_table_structure("table_name")
     ```

2. **SQL 构造脚本**:
   - 脚本路径: `scripts/allocation_utils.py`
   - 功能: 动态生成 SQL 查询语句。
   - 示例调用:
     ```python
     from skills.finance.scripts.allocation_utils import generate_alloc_sql
     sql_query = generate_alloc_sql("table_name", "field_name", "condition")
     ```

3. **SQL 执行脚本**:
   - 脚本路径: `scripts/sql_query.py`
   - 功能: 执行 SQL 查询并返回结果。
   - 示例调用:
     ```python
     from skills.finance.scripts.sql_query import run_sql_query
     result = run_sql_query("SELECT * FROM table_name")
     ```

1. **解析意图**:
   包括关键字提取和表映射，分析问题是否与财务相关。
   示例：对于提问“白领数分摊服务有哪些？”，技能会判断“分摊”和“服务”关键词，找到相关数据库表及字段。

2. **查询表结构信息**:
   动态获取数据库表结构和示例数据，确保生成的 SQL 语句字段正确。
   示例：查询表 `SSME_FI_InsightBot_CostDataBase` 的字段和示例数据。

3. **生成 SQL**:
   调用模板生成 SQL 查询。例如，针对表 `SSME_FI_InsightBot_CostDataBase` 构造分摊场景的 SQL 查询。

4. **执行查询**:
   执行 SQL，返回查询结果给用户。

5. **分析结果并返回**:
   对查询结果进行分析，生成用户可读的结论。
   示例：对分摊金额进行汇总，计算变化率并生成报告。

## 完整流程强制约束（必须走完）

- 必须完整执行以下链路后才能回答：
  1. 解析意图
  2. 查询表结构信息
  3. 术语归一化与字段映射
  4. 生成 SQL
  5. 校验 SQL（语法、字段值、业务口径）
  6. 执行 SQL 并获取结果
  7. 基于结果生成结论
- 任一步骤失败都不得直接给业务结论，必须返回失败原因并继续修正 SQL 后重试。
- 若 SQL 未执行成功（无结果、执行报错、结果结构异常），回答中必须明确标注“流程未完成”，禁止输出确定性结论。
- 最终回答必须可追溯到查询结果：
  - 需要引用关键结果字段或汇总值；
  - 对比类问题必须同时给出差值与变化率；
  - 不允许仅基于经验或常识给出结论。

## 示例场景

用户问题示例：

- “白领数分摊服务有哪些？”
- “今年 HR 部门成本预算与去年实际的趋势对比如何？”

技能会基于问题动态生成 SQL 并返回结果。
