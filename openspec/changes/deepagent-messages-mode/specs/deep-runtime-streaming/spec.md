## ADDED Requirements

### Requirement: Token-level streaming via messages mode
The DeepAgentRuntime SHALL support true token-level streaming by using `stream_mode="messages"` instead of `stream_mode="updates"`.

#### Scenario: Real-time text streaming
- **WHEN** user sends a message to the agent
- **THEN** the system SHALL stream individual tokens as they are generated by the LLM
- **AND** each token SHALL be sent to the frontend as a separate `text_delta` event
- **AND** the frontend SHALL display tokens immediately without waiting for the complete response

## MODIFIED Requirements

### Requirement: Event conversion from messages stream
The StreamEventConverter SHALL convert LangGraph message stream events into AgentEvent format suitable for frontend consumption.

#### Scenario: Converting AIMessageChunk to text_delta
- **WHEN** LangGraph yields an `("messages", AIMessageChunk)` tuple
- **THEN** the converter SHALL extract the `content` field from the chunk
- **AND** yield an `AgentEvent(type="text_delta", data=content)` event

#### Scenario: Detecting tool calls in message stream
- **WHEN** an AIMessageChunk contains `additional_kwargs.tool_calls`
- **THEN** the converter SHALL yield a `tool_start` event with tool call details
- **AND** continue streaming until tool execution completes

#### Scenario: Handling tool result messages
- **WHEN** LangGraph yields a ToolMessage in the stream
- **THEN** the converter SHALL yield a `tool_end` event
- **AND** include the tool execution result in the event data

#### Scenario: Completion signaling
- **WHEN** the LLM finishes generating the final response (stream ends)
- **THEN** the converter SHALL yield a `complete` event
- **AND** the accumulated full message SHALL be saved to the dialog history
