1. 重构前后对比

❌ 重构前

  • Hook 系统深度耦合
  • 业务逻辑混杂在路由层
  • 循环导入问题严重
  • 全局状态难以追踪
  • 扩展困难，调试复杂

✅ 重构后

  • Facade 门面统一入口
  • 依赖注入解耦模块
  • EventBus 异步通信
  • 清晰的分层架构
  • 支持多协议 (HTTP/WS/CLI)

3. 核心代码示例

Facade - AgentEngine

🏛️
core/engine.py
class AgentEngine:
    """Agent 核心引擎 - Facade 模式"""
    
    def __init__(self, config: dict):
        self._event_bus = EventBus()
        
        # 初始化 Manager (依赖注入)
        self._dialog_mgr = DialogManager(
            event_bus=self._event_bus,
            state_manager=self._state_mgr,
            config=config
        )
        self._tool_mgr = ToolManager(
            event_bus=self._event_bus,
            config=config
        )
        # ... 其他 Manager
    
    # 代理方法
    async def create_dialog(self, user_input: str) -> str:
        return await self._dialog_mgr.create(user_input)
    
    async def send_message(self, dialog_id: str, message: str):
        async for chunk in self._dialog_mgr.send_message(dialog_id, message):
            yield chunk

EventBus - 事件总线

📡
runtime/event_bus.py
class EventBus:
    """统一事件总线 - 发布订阅模式"""
    
    def subscribe(self, callback, event_types=None, dialog_id=None):
        # 支持过滤订阅
        sub_id = ++self._next_id
        self._subscribers.append((sub_id, callback, {
            'types': event_types,
            'dialog_id': dialog_id
        }))
        return lambda: self._unsubscribe(sub_id)
    
    def emit(self, event, session_path=None):
        # 异步分发给所有匹配的订阅者
        for callback, filter in self._subscribers:
            if self._matches_filter(event, filter):
                asyncio.create_task(callback(event))

4. 模块间依赖关系

interfaces/ (HTTP/WebSocket/CLI)
仅依赖 core.engine 和 runtime.events
▼
core/ (Engine + Managers)
依赖 runtime 和 infrastructure
▼
runtime/ (EventBus + Stream)
仅依赖 core.models
▼
infrastructure/ (Config/Logging/DB)
最底层，被所有层依赖

依赖规则

层级 可以依赖 禁止依赖
interfaces/ core.engine, runtime.events core.managers (内部)
core/ runtime, infrastructure interfaces (循环)
runtime/ core.models core.engine
infrastructure/ 无 (纯底层) 任何上层

5. 迁移路径

阶段 1: 创建新结构

# 创建新目录结构
mkdir -p core/managers core/models core/stores
mkdir -p interfaces/http/routes interfaces/websocket/handlers
mkdir -p runtime/events runtime/stream/parsers

阶段 2: 实现核心组件

  1. 实现 runtime/event_bus.py (事件总线)
  2. 实现 core/engine.py (Facade)
  3. 逐个迁移 Manager (dialog → tool → skill → state → memory)

阶段 3: 迁移接口层

  1. 实现 interfaces/http/server.py
  2. 迁移路由，将业务逻辑移到 Manager
  3. 实现 WebSocket 处理器

阶段 4: 切换入口

# main.py - 新入口
from core.engine import AgentEngine
from interfaces.http.server import create_app

async def main():
    config = load_config()
    engine = AgentEngine(config)
    await engine.startup()
    
    app = create_app(engine)
    await run_server(app)

6. 关键改进总结

方面 重构前 重构后 收益
架构模式 Hook 系统，深度耦合 Facade + Manager 职责清晰，易于测试
模块通信 直接调用，回调嵌套 EventBus 发布订阅 彻底解耦，可扩展
依赖管理 循环导入，全局状态 构造函数注入 可测试，可替换
接口层 业务逻辑混杂在路由 纯协议转换 支持多协议
数据流 双向混乱 单向: Engine → Event → Interface 可预测，易调试
